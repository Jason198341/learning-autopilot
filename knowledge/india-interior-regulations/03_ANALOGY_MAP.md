## 🎯 Primary Analogy: 건축 허가와 비슷해요

### This concept is like...

```
[인도 자동차 법규] ≈ [건물 건축 허가]
```

### 왜 이 비유가 맞을까요?

```
건축 허가 절차                      자동차 형식 승인 (TAC)
═══════════════                    ═══════════════════════

1. 건축법 확인                      1. CMVR/AIS 확인
   · 용도 지역                         · 차량 카테고리
   · 건폐율/용적률                     · 적용 법규
        │                                  │
        ↓                                  ↓
2. 설계 도면 제출                   2. 기술 문서 제출
   · 건축 설계도                       · 도면/사양서
   · 구조 계산서                       · 시험 성적서
        │                                  │
        ↓                                  ↓
3. 심사 & 현장 점검                 3. 시험 & 공장 심사
   · 구청 심사                         · ARAI 시험
   · 현장 확인                         · COP 심사
        │                                  │
        ↓                                  ↓
4. 건축 허가증 발급                 4. TAC 발급
   · 착공 가능                         · 판매 가능

공통점:
· 법적 요건 충족 확인
· 안전 기준 검증
· 인증 없이는 불법
```

## 🚦 삼소구 = 운전면허 필수 서류

### 비유: 운전 시 필수 구비 서류

```
운전면허 관련 필수 서류             차량 필수 장비 (삼소구)
═══════════════════════           ═══════════════════════

운전 시 반드시 휴대:               차량에 반드시 구비:

┌─────────────────┐               ┌─────────────────┐
│ 1. 운전면허증   │               │ 1. 삼각대       │
│    (신분 증명)  │               │    (사고 표시)  │
├─────────────────┤               ├─────────────────┤
│ 2. 자동차등록증 │               │ 2. 소화기       │
│    (차량 증명)  │               │    (화재 대응)  │
├─────────────────┤               ├─────────────────┤
│ 3. 보험증서    │               │ 3. 구급상자     │
│    (사고 대비)  │               │    (부상 대응)  │
└─────────────────┘               └─────────────────┘

없으면? → 범칙금!                  없으면? → 벌금/인증 불가!

"면허증 없이 운전 못하듯,
 삼소구 없이 인도 판매 못한다!"
```

## 🔄 Pattern Recognition

### 패턴 1: "피라미드 구조"

```
모든 국가의 자동차 법규는 피라미드:

                    ▲ 법률 (Act)
                   ╱ ╲
                  ╱   ╲
                 ╱ 규칙 ╲ (Rules)
                ╱───────╲
               ╱  표준   ╲ (Standards)
              ╱───────────╲
             ╱    규격     ╲ (Specifications)
            ╱───────────────╲

인도:  MVA → CMVR → AIS → IS
한국:  자동차관리법 → 안전기준 → KMVSS
EU:    Framework Directive → UN-R → 상세규격
```

### 패턴 2: "시험-인증-감시"

```
모든 인증 시스템의 공통 흐름:

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   개발 → 시험 → 인증 → 생산 → 감시                           │
│    │      │      │      │      │                           │
│    ↓      ↓      ↓      ↓      ↓                           │
│   설계   샘플   승인   양산   사후                            │
│   검토   테스트  발급   적합성  관리                          │
│                  (TAC)  (COP)  (시장감시)                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 💡 Metaphors

### Technical Metaphor

> "인도 자동차 법규는 **입국 심사**와 같다. CMVR이 비자 정책이라면, AIS는 심사 기준이고, ARAI는 입국 심사대이며, TAC는 입국 스탬프다. 스탬프 없이는 인도 시장에 들어갈 수 없다."

### Everyday Metaphor

> "삼소구는 **비상용 생존 키트**와 같아요. 비행기에 구명조끼와 산소마스크가 있듯이, 인도에서 판매하는 차에는 삼각대, 소화기, 구급상자가 있어야 해요. 평소엔 안 쓰지만, 비상시엔 생명을 구해요."

### Teaching Metaphor

> "AIS 시험은 **운전면허 시험**과 같아요. 필기(서류 심사)도 보고, 기능(샘플 시험)도 보고, 주행(양산 적합성)도 봐야 해요. 한 과목이라도 떨어지면 면허(TAC)를 못 받아요!"

## 🧩 Where Analogies Break

### 비유가 안 맞는 부분

| 비유 | 한계점 |
|------|--------|
| **건축 허가** | 차량은 대량 생산 (1회 허가로 다량 판매) |
| **학교 시험** | TAC는 갱신 필요 (영구 자격 아님) |
| **운전면허 서류** | 삼소구는 물리적 장비 (서류 아님) |

### 인도 법규만의 특별한 점

```
인도 자동차 법규의 특징:

1. 급격한 변화
   → 매년 새로운 규제 추가
   → OEM들의 빠른 대응 필요

2. 인도 고유 요건
   → 고온/먼지 환경 고려
   → 저가 차량 특화 기준
   → 인도 도로 조건 반영

3. 단계적 적용
   → Phase 1, 2, 3... 순차 도입
   → 차종별 다른 시행 시기
```

## 🧠 Transfer Learning

### 인도 법규에서 배우는 범용 원리

```
1. 법규 구조 이해
   ─────────────────
   "어느 나라든 피라미드 구조"
   → 법률 → 규칙 → 표준 → 규격
   → 한 번 배우면 다른 나라에도 적용

2. 인증의 본질
   ─────────────────
   "시험 → 승인 → 감시"
   → 사전 검증 + 사후 관리
   → 소비자 보호가 목적

3. 지역화의 중요성
   ─────────────────
   "각 나라마다 고유 요건 있음"
   → 기후, 도로, 문화 반영
   → 글로벌 스탠다드 + 로컬 요건
```
