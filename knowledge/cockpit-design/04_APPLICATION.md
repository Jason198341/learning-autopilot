## 🚙 Real-World Use Cases

### Use Case 1: SUV vs 세단 IP 차이

**Context**: 같은 브랜드인데 SUV와 세단의 IP가 왜 다를까요?

```
세단 (Sedan)                      SUV
═══════════                       ═══

앉는 높이: 낮음                    앉는 높이: 높음
H-Point: 낮음                     H-Point: 높음
    │                                 │
    ▼                                 ▼

┌─────────────────┐              ┌─────────────────┐
│    앞유리 각도    │              │    앞유리 각도    │
│      ╲          │              │        │        │
│       ╲ 완만    │              │        │ 가파름  │
│        ╲        │              │        │        │
│    IP 높이 낮음  │              │    IP 높이 높음  │
│                 │              │                 │
│  ※ 스포티한 느낌 │              │  ※ 시원한 느낌   │
└─────────────────┘              └─────────────────┘
```

**How it's applied:**

| 항목 | 세단 | SUV | 이유 |
|------|------|-----|------|
| IP 높이 | 낮음 | 높음 | H-Point 차이 |
| 클러스터 각도 | 위로 향함 | 정면 | 눈높이 맞춤 |
| 콘솔 폭 | 좁음 | 넓음 | 어깨 위치 |
| 버튼 위치 | 낮음 | 높음 | 리치 영역 |

### Use Case 2: 전기차 칵핏 변화

**Context**: 내연기관차 → 전기차로 바뀌면서 칵핏이 어떻게 달라졌을까요?

```
내연기관차                          전기차
══════════                         ═══════

┌─────────────────┐               ┌─────────────────┐
│   [클러스터]     │               │   [클러스터]     │
│   RPM 타코미터  │               │   없거나 최소화  │
│   (엔진 회전수)  │               │   (모터는 RPM    │
│                 │               │    볼 필요 없음) │
└─────────────────┘               └─────────────────┘

┌─────────────────┐               ┌─────────────────┐
│   [센터]        │               │   [센터]        │
│   기어봉 (큼)   │               │   버튼/다이얼    │
│   변속기 터널   │               │   플랫 플로어   │
│                 │               │   (터널 없음)   │
└─────────────────┘               └─────────────────┘

┌─────────────────┐               ┌─────────────────┐
│   [정보]        │               │   [정보]        │
│   연료 게이지   │               │   배터리 %      │
│   엔진 온도     │               │   주행 가능거리  │
│                 │               │   충전 상태     │
└─────────────────┘               └─────────────────┘
```

**전기차 칵핏의 새로운 트렌드:**

1. **미니멀리즘**: 필요 없는 건 과감히 삭제
2. **대형 스크린**: 물리 버튼 → 터치로 통합
3. **플랫 플로어**: 변속기 터널 사라짐 → 수납 공간 확보
4. **새로운 정보**: 배터리, 회생제동, 충전소 안내

### Use Case 3: 고급차 vs 양산차 차별화

**Context**: 비싼 차는 칵핏이 뭐가 다를까요?

```
양산차 (2,000만원)                 고급차 (8,000만원)
═══════════════                   ═══════════════

재질:                              재질:
· 하드 플라스틱                    · 소프트 패드 + 가죽
· 직물 시트                        · 천연 가죽 시트
· 도금 버튼                        · 알루미늄/우드 버튼

조작계:                            조작계:
· 수동 조절 (대부분)               · 전동 조절 (전부)
· 일반 오디오                      · 프리미엄 오디오 (B&O, 하만)
· 수동 에어컨                      · 오토 에어컨 + 온열/통풍

디스플레이:                        디스플레이:
· 7인치 (기본)                     · 12인치+ (커브드)
· 아날로그 클러스터                 · 디지털 클러스터
· 없음                             · HUD

기능:                              기능:
· 기본 경고                        · 360도 뷰
· 후방 카메라                      · 자동 주차
· 크루즈                          · 레벨 2 자율주행
```

**고급차 칵핏 차별화 포인트:**

| 감각 | 양산차 | 고급차 |
|------|--------|--------|
| 시각 | 기능적 | 프리미엄 소재/마감 |
| 촉각 | 하드 플라스틱 | 소프트 터치, 가죽 |
| 청각 | 일반 소리 | 도어 닫는 소리까지 튜닝 |
| 후각 | 없음/플라스틱 냄새 | 천연가죽 향, 무취 |

## 📝 Practice Problems

### Level 1: Basic (기초)

**Problem 1.1**: 클러스터 위치
> 클러스터(계기판)는 왜 핸들 뒤에 있을까요?

<details>
<summary>💡 정답 보기</summary>

```
이유:

1. 시선 이동 최소화
   핸들 뒤 = 전방 도로에서 가장 가까움
   → 눈을 크게 움직이지 않아도 됨

2. 가림 방지
   핸들 위로 보이도록 배치
   → 손이 가려도 정보 보임

3. 조작 분리
   클러스터 = 정보 (보기만)
   핸들 = 조작 (만지기만)
   → 헷갈리지 않음
```

</details>

> 자동차 경고등이 빨간색/주황색/녹색인 이유는?

<details>

### Level 2: Intermediate (중급)

**Problem 2.1**: 버튼 배치 분석
> 아래 에어컨 조작부 중 어떤 게 더 좋은 설계일까요?

```
디자인 A:                          디자인 B:
┌─────────────────┐               ┌─────────────────┐
│  🌡️ ❄️ 💨       │               │     [25°C]      │
│  버튼 버튼 버튼  │               │   [-] [+]       │
│                 │               │   💨 AUTO 💨    │
│    [←] [→]      │               │   [OFF]         │
└─────────────────┘               └─────────────────┘
```

<details>
<summary>💡 정답 보기</summary>

```
디자인 B가 더 좋아요!

디자인 A의 문제:
· 아이콘만으로 기능 파악 어려움
· 현재 상태(온도) 확인 안 됨
· 버튼이 비슷해서 헷갈림

디자인 B의 장점:
· 현재 온도 숫자로 표시
· +/- 직관적인 조작
· AUTO 모드로 간편 사용
· OFF 버튼 분리

좋은 조작부 설계 원칙:
1. 현재 상태가 보여야 함
2. 조작 방향이 명확해야 함
3. 자주 쓰는 건 크고 눈에 띄게
```

</details>

## ✅ 칵핏 설계 체크리스트

### 기획 단계
```
□ 타겟 고객 정의 (연령, 성별, 용도)
□ 차종 특성 반영 (SUV/세단/스포츠)
□ 경쟁차 벤치마킹
□ 규제 요건 확인 (에어백, 시야)
```

### 레이아웃 단계
```
□ H-Point 설정 완료
□ 5%ile~95%ile 리치 검증
□ 시야 영역 확보 (A-pillar 사각 체크)
□ 에어백 전개 공간 확보
□ 페달 거리 조절 범위 설정
```

### 조작계 단계
```
□ 자주 쓰는 기능 = Zone A 배치
□ 블라인드 조작 가능 여부
□ 촉감 피드백 확인
□ 오조작 방지 설계
□ 좌핸들/우핸들 대응
```

### 디스플레이 단계
```
□ 정보 우선순위 결정
□ 폰트/색상 시인성 검증
□ 글레어 방지 각도
□ 주야간 밝기 조절
□ 반응 속도 (100ms 이내)
```

### 검증 단계
```
□ 인간공학 시뮬레이션 (DHM)
□ 클레이 모델 평가
□ 시작차 조작성 평가
□ 운전 시뮬레이터 테스트
□ 실차 주행 테스트
```

## ⚠️ Common Mistakes

### 신입사원이 자주 하는 실수

| 실수 | 왜 문제인가 | 올바른 접근 |
|------|------------|------------|
| "예쁘니까 이 위치!" | 사용성 무시 | 인간공학 먼저, 디자인 나중 |
| "평균 치수로 설계" | 5%ile, 95%ile 불편 | 전 범위 검증 |
| "버튼 많으면 좋지" | 복잡 = 오조작 | 기능 통합, 심플하게 |
| "큰 화면이 좋지" | 시선 분산 | 적정 크기, 정보 계층화 |
| "터치가 최신 트렌드" | 블라인드 조작 불가 | 물리 버튼 병행 |
| "검은색이 고급스럽지" | 글레어, 지문 | 환경 고려한 색상 |

### 피해야 할 안티패턴

```
❌ 안티패턴 1: "기능 욕심"
   모든 기능을 칵핏에 넣으려 함
   → 복잡해져서 아무도 못 씀

✓ 해결: 80/20 법칙
   20%의 기능이 80%의 시간 사용됨
   → 핵심 기능만 전면에

❌ 안티패턴 2: "최신 기술 맹신"
   터치, 제스처, 음성 등 신기술만
   → 익숙한 게 더 빠를 수 있음

✓ 해결: 익숙함 + 혁신 밸런스
   핵심 기능은 물리 버튼
   부가 기능은 신기술

❌ 안티패턴 3: "자동차 안에서만 생각"
   이 차만 보고 설계
   → 다른 차에서 온 고객이 헷갈림

✓ 해결: 업계 관행 존중
   방향지시등 = 왼쪽 레버 (글로벌 표준)
   갑자기 바꾸면 혼란
```
