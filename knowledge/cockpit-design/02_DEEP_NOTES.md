## 🧬 Layer 1: 인간공학 기초

### H-Point가 뭐예요?

> **H-Point** = Hip Point = 엉덩이 기준점
> 모든 칵핏 설계의 **시작점**이에요!

```
왜 중요할까요?

사람마다 키가 다르잖아요.
그런데 차는 하나예요.
어떻게 모든 사람에게 맞출 수 있을까요?

답: "엉덩이 위치를 기준으로 삼는다!"

H-Point를 정하면:
→ 눈 위치 (아이포인트) 가 정해지고
→ 손 위치 (리치) 가 정해지고
→ 발 위치 (페달 거리) 가 정해집니다!
```

```
              [ 운전자 측면 ]

              ○ ← 머리 (Head)
             /│
            / │
           /  │   ← 눈 (Eye Point)
              │      여기서 클러스터를 봐요
           ───┼───
              │
              ●──── H-Point (엉덩이!)
             /│      ← 모든 치수의 기준점
            / │
           /  │
          ●   │     ← 무릎 (Knee)
         /    │
        /     │
       ●──────┴──── 발 (Foot)
                    페달을 밟아요
```

### 95%ile 남성? 5%ile 여성?

> 💡 **%ile (퍼센타일)** = 100명 중 몇 번째인가?

```
키 순서로 100명을 세웠을 때:

5%ile (5번째) = 작은 쪽에서 5번째
             → 작은 여성 대표

50%ile (50번째) = 딱 중간
               → 평균적인 사람

95%ile (95번째) = 큰 쪽에서 5번째
               → 큰 남성 대표
```

### 왜 5%ile 여성 ~ 95%ile 남성을 기준으로?

```
전체 인구의 90%를 커버하려면:

5%ile 여성 ─────────────────────────────── 95%ile 남성
    │                                         │
    │     ← 이 범위 안에 90%의 사람이 있어요 →    │
    │                                         │
 작은 손                                    큰 손
 짧은 팔                                    긴 팔
 낮은 눈                                    높은 눈

설계할 때:
- 버튼은 5%ile도 닿게 (팔 짧은 사람)
- 공간은 95%ile도 들어가게 (몸 큰 사람)
```

### 실제 수치 (한국인 기준)

| 항목 | 5%ile 여성 | 50%ile 평균 | 95%ile 남성 |
|------|-----------|------------|-------------|
| 키 | 152cm | 166cm | 180cm |
| 앉은 눈 높이 | 70cm | 77cm | 84cm |
| 팔 뻗는 거리 | 62cm | 70cm | 78cm |
| 어깨 너비 | 36cm | 42cm | 48cm |

### 글레어(반사) 방지 설계

> 🌞 햇빛이 IP에 반사되면 앞이 안 보여요!

```
글레어가 생기는 경우:

    태양 ☀️
         ╲
          ╲  빛
           ╲
            ╲
    ─────────╲───────────── 앞유리
              ╲
               ╲
         IP 표면 ════════  ← 여기서 반사!
                    ╲
                     ╲
                      ● 운전자 눈 ← 눈이 부셔요!
```

**해결책:**

| 방법 | 설명 | 적용 |
|------|------|------|
| **각도 조절** | IP 상단을 뒤로 기울임 | 반사광이 위로 가게 |
| **무광 표면** | 광택 없는 재질 | 난반사로 분산 |
| **어두운 색** | 검정/짙은 회색 | 반사율 감소 |
| **시보 처리** | 표면에 미세 요철 | 빛을 흩뜨림 |

**전문가 수치:**
- IP 상단 각도: 수평에서 10~15° 기울임
- 표면 광택도: 60° 광택 2GU 이하
- 반사율: 5% 이하 권장

### 화면 각도의 비밀

```
왜 화면을 기울여야 할까요?

수직 화면의 문제:
         │
         │ ← 화면
         │
         ● ← 눈

문제: 반사광이 눈에 들어옴!

기울인 화면의 해결:
         ╲
          ╲ ← 화면 (10~15° 기울임)
           ╲
            ● ← 눈

해결: 반사광이 위로 가서 눈에 안 들어옴!
```

**전문가 수치:**
- 화면 기울기: 수직에서 10~20° 뒤로
- 눈-화면 거리: 500~700mm
- 화면 중심 각도: 아이포인트에서 20~30° 아래

### 블라인드 조작 설계

> 💡 **블라인드 조작** = 눈으로 안 보고도 손으로 조작

```
블라인드 조작이 가능하려면?

1. 촉감으로 구분
   ┌─────┐  ┌─────┐  ┌─────┐
   │ ○○○ │  │ ▭▭▭ │  │ ◇◇◇ │
   │돌기형 │  │ 홈형 │  │ 요철형│
   └─────┘  └─────┘  └─────┘
   "만져보면 뭔지 알겠다"

2. 위치로 구분
   에어컨 = 항상 같은 위치
   볼륨 = 항상 같은 위치
   "위치 외우면 안 봐도 됨"

3. 피드백
   눌렀을 때: 딸깍 소리 / 진동
   "눌렸는지 느껴져야 함"
```

### 스위치 배치 원칙

```
스티어링 스위치 배치:

         ┌─────────────────────┐
         │                     │
    ╭────┤                     ├────╮
   ╱     │     [에어백]         │     ╲
  │  [A] │      ┌───┐          │ [B]  │
```
